# Assignment: product-behavior

## Papel: Visão de Produto e Priorização

### Contexto

O roadmap principal (`14-roadmap.md`) define 6 fases com 40 submódulos. O roadmap técnico (`roadmap.json`) detalha as primeiras fases.

O protótipo React serve como referência do produto desejado. O Flutter precisa alcançar e superar essa referência.

### Branches

```
feature/* (revisão contínua em paralelo)
```

### Skills

Nenhum skill específico — trabalho de produto.

### Tarefas Contínuas

#### 1. Validar Roadmap

Verificar se o roadmap atual (`14-roadmap.md` e `roadmap.json`) está alinhado com:
- O que o protótipo React já implementa
- O que está sendo construído nas fases atuais
- Feedback implícito (gaps descobertos durante implementação)

#### 2. Definir Critérios de Aceite

Para cada fase, definir:
- O que significa "pronto" para essa fase
- Critérios de aceite objetivos
- Checklist de review de produto

#### 3. Priorização de Backlog

Manter backlog de features não-planejadas que surgirem:
- Features novas descobertas durante implementação
- Melhorias na experiência do usuário
- Correções de usabilidade

#### 4. Consistência de Produto

Garantir que:
- Nomenclatura é consistente entre telas
- Fluxos de usuário são consistentes
- A "estrela do norte" do produto está sendo respeitada

### Resultados Esperados

- [ ] Roadmap validado e atualizado após cada fase
- [ ] Critérios de aceite definidos para cada fase
- [ ] Backlog de produto mantido
- [ ] Consistência de produto preservada

### Nota

Este agente trabalha em paralelo com a implementação — não bloqueia nem é bloqueado por outras fases.
