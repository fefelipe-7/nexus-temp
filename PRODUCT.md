# Product

## Register

product

## Users

Adultos que buscam autoconhecimento e melhoria contínua via rastreamento diário de saúde, mente, trabalho, finanças e vida social. Usam o app no celular, em momentos de transição (ao acordar, durante refeições, antes de dormir). Não são power users de produtividade — querem simplicidade e consistência.

## Product Purpose

Nexus é um sistema pessoal de registro e análise que conecta dados diários (sono, humor, refeições, treinos, gastos, tarefas) a insights acionáveis. Funciona 100% offline-first, sem dependência de nuvem. O objetivo é revelar padrões que o usuário não veria sozinho — correlações entre métricas, tendências ao longo do tempo, recomendações contextuais.

## Brand Personality

Calmo, confiável, inteligente. O tom é de um assistente discreto: presente quando necessário, silencioso quando não. Nunca ansioso, nunca gamificado demais. A interface deve passar segurança — os dados do usuário estão seguros, as correlações são precisas, o sistema não desaparece.

## Anti-references

- Apps de produtividade genéricos com fundo roxo-para-azul gradient (Todoist, Any.do)
- Interfaces "gamificadas" com badges, streaks, confetes (Duolingo quando excessivo)
- Dashboards corporativos pesados (Jira, Monday.com)
- O cream/sand/beige background que domina 90% dos templates AI atuais

## Design Principles

1. **A ferramenta desaparece.** O usuário vem para registrar ou consultar, não para admirar a UI. Cada pixel ou atrapalha ou ajuda.
2. **Dados primeiro, decoração depois.** Cor e motion servem para comunicar estado e hierarquia, não para enfeitar.
3. **Consistência é confiança.** Mesmo botão, mesmo card, mesmo comportamento em todas as telas. Surpresa é para momentos específicos.
4. **Calma visual.** Espaço para respirar, tipografia legível, contraste suficiente. O app não compete com a atenção do usuário — ele a organiza.
5. **Offline-first com dignidade.** A interface nunca depende de rede. Loading states, empty states e erros são cidadãos de primeira classe.

## Accessibility & Inclusion

- WCAG AA como piso (contraste 4.5:1 body, 3:1 large text)
- `prefers-reduced-motion` respeitado em todas as animações
- `prefers-color-scheme` suportado (dark mode desde o lançamento)
- Fontes com fallback para sistema, sem dependência de web font para renderizar
